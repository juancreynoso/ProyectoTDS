Análisis Léxico

Descripción

El análisis léxico se lleva a cabo en la primera etapa del compilador y se encarga de transformar la entrada de texto en una secuencia de tokens. Estos tokens son unidades atómicas que alimentan al parser.

Herramientas

  • Flex para el scanner
  • Integración con Bison a través de yylval

Ejemplo de Definición de Reglas

      "program"   { return PROGRAM; }
      "extern"    { return EXTERN; }
      "then"      { return THEN; }
      "while"     { return WHILE; }

      ">"         { return GT; }
      "||"        { return OR; }

      "("         { return PAREN_L; }
      "{"         { return LLAVE_L; }

      {digit}+    { return NUM; }
      {ID}        { return ID; }

      [ \t\n]+    { /* ignorar espacios, tabs y saltos de línea */ }

      ";"         { return PYC; }

      "//".*       { /* ignorar comentario de línea */ }
El lexer reconoce cadenas de texto específicas y las devuelve como tokens, quienes serán utilizados en la proxima etapa por el analizador sintáctico.

Dificultades Encontradas y Resolución

Durante la implementación del lexer se presentaron algunos desafíos, entre ellos el tratamiento de los comentarios. En un principio intentamos reconocerlos y devolver tokens, tanto para los comentarios de una sola línea como para los multilínea. Sin embargo, dado que los comentarios no poseen valor gramatical dentro del lenguaje, la solución adoptada fue reconocerlos únicamente para descartarlos, evitando así la generación de cualquier token asociado.
